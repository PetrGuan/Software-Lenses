# Meadows Mindset

> "Small, well-placed interventions can outperform massive local effort."

**Category:** Strategy | **Skill:** `/meadows`

## When to Use

- Teams work hard but outcomes do not improve
- Recurring incidents persist across local fixes
- Metric gaming distorts behavior

## Core Move

Locate leverage in feedback loops, information flows, rules, and goals, then change the highest-impact point first.

## Diagnostic Questions

- Which loop currently amplifies failure?
- Which metric drives the wrong local behavior?
- Which rule change would alter system behavior globally?

## Output Format

```
System Map:
  [actors, loops, constraints]

Current Leverage Ranking:
  [high -> low impact interventions]

Top Intervention:
  [first change with expected impact]

Validation Plan:
  [how to detect real improvement]
```

## Anti-Patterns This Catches

- Applying maximum effort at the lowest-leverage point in the system
- Optimizing metrics that reward local behavior while the system degrades globally
- Ignoring reinforcing loops that amplify small problems into large ones
- Treating all interventions as equal instead of ranking by systemic impact
- Changing parameters when the real leverage is in changing rules or goals

## Pairs Well With

- **Deming** — Deming maps the system that produces failures, Meadows ranks where to intervene in that system
- **Grothendieck** — Grothendieck restructures the model, Meadows identifies which structural change has the highest leverage
- **Ostrom** — Meadows finds the leverage point, Ostrom turns it into a durable governance rule
- **Wiener** — Wiener models feedback and control dynamics, Meadows uses that model to locate intervention points

## When Not to Over-Apply

- The system is well-understood and the fix is a straightforward local correction
- Mapping leverage points would delay an urgent tactical response
- The team lacks the authority or political capital to change the high-leverage point identified
